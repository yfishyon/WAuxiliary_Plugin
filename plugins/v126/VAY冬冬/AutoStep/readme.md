# 微信步数自动增加插件（AutoStep）

## 作者信息
- 模块原作者：Wukon
- 修改：VAY冬冬

## 插件概述
AutoStep 在可配置活动时段内自动调整微信步数，支持时间触发、消息触发、保底补齐与目标分配策略。

### 核心能力
1. **时间步数**：在活动时间段内每分钟自动增加步数，支持每分钟步数范围配置。
2. **消息步数**：消息触发增加步数，低步数区间增幅更高。
3. **保底步数**：22:50 检查，低于保底值则补齐。
4. **时间目标**：可设置截止前需达到的目标步数（线性/指数分配）。
5. **上限后自动暂停**：达到最大步数后可自动跳过插件增长与上传，让微信自然增长。

### 消息触发增幅规则
- 4000 步以下：3 倍
- 4000-8000 步：2 倍
- 8000-16000 步：1 倍
- 16000 步以上：0.5 倍

## 使用方式
1. 可通过命令控制功能（见下方命令列表）。
2. 可通过 UI 面板进行可视化配置（推荐），在聊天输入框发送以下任一关键词打开：
   - `步数设置`
   - `步数管理`
   - `自动步数`
3. 每日 0 点自动重置插件内部统计。
4. 功能状态自动保存，重启后保持上次配置。

## 命令（即时生效）
- `/时间步数开` - 启用时间自动增加
- `/时间步数关` - 关闭时间自动增加
- `/消息步数开` - 启用消息触发增加
- `/消息步数关` - 关闭消息触发增加
- `/步数状态` - 显示当前步数与关键状态（含时间目标）
- `/步数状态all` - 显示完整状态（含分配策略、目标等）
- `/改步数 N` - 手动设置当前步数为 N
- `/最大消息步数 N` - 设置消息触发的目标上限
- `/分钟步数范围 min-max` - 设置时间模式下每分钟步数范围
- `/保底步数 N` - 设置每日保底步数（0 表示关闭）
- `/时间步数目标 N` - 设置截止前目标步数（0 表示取消）
- `/分配策略 线性|指数` - 切换时间目标分配策略

示例：
- `/时间步数目标 20000`
- `/分配策略 指数`
- `/分钟步数范围 4-12`

## UI 可配置项
- 功能开关：时间步数、消息步数、日志输出、测试模式、上限后自动暂停
- 参数设置：最大步数、每分钟步数范围、最大消息步数、保底步数、时间步数目标、分配策略、活动时间段（`HH:mm-HH:mm`）
- 快捷操作：手动改步数、重置今日步数

## 注意事项
- 自动步数默认活动时间段为 `07:00-22:50`，可在 UI 中修改；测试模式可忽略该时段限制。
- 保底步数在 22:50 执行。
- 当“上限后自动暂停”开启且达到最大步数后，插件会跳过后续步数增长与上传。
- 若当天实际步数高于插件步数，以微信侧实际同步为准。
- 时间模式依赖微信后台存活，后台被清理会影响及时更新。
- 消息模式依赖消息触发，后台无消息时会减少触发频率。
- 日志输出：运行过程写入日志文件(Android/media/com.tencent.mm/WAuxiliary/Plugin/AutoStep/autostep_log.txt)
- 测试模式：在非活跃时段也允许执行时间/消息触发逻辑”
